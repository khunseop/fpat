# Policy Deletion Processor

추출된 방화벽 데이터를 바탕으로 정책 삭제 영향도를 분석하고 정리 프로세스를 지원하는 모듈입니다.

## 🚀 주요 기능

- **신청 정보 연동**: 정책 설명(Description) 및 외부 데이터로부터 신청번호(MIS ID) 및 사용자 정보 매칭.
- **자동 예외 필터링**: `fpat.yaml`에 설정된 기간제 및 패턴 기반 예외 정책 자동 분류.
- **중복 및 미사용 분석**: 정책간 포함 관계 및 히트카운트 기반의 삭제 대상 선정.
- **공지 자동화**: 분석 결과에 따라 사용자 공지용 및 장비 삭제용 파일 분리 생성.

## 🔧 CLI 사용법 (추천)

통합 CLI를 통해 대화형 입력 없이 모든 분석 단계를 자동화할 수 있습니다.

```bash
# 기본 구조
python -m fpat process --task [작업번호] --files [입력파일리스트]
```

### 작업 번호 안내 (Task IDs)

| ID | 작업 명칭 | 설명 |
|:---|:---|:---|
| **1** | 신청번호 파싱 | Description에서 신청 정보를 추출하여 정책 파일에 반영 |
| **2** | 신청 ID 추출 | 정책 파일 내의 고유 신청 ID 리스트를 엑셀 시트별로 저장 |
| **3** | MIS ID 보강 | 별도 CSV 파일로부터 매칭되는 MIS ID 추가 |
| **4** | GSAMS 정보 가공 | 외부 신청정보 원본을 표준 분석 포맷으로 변환 |
| **5** | 신청 정보 매칭 | 가공된 신청정보를 정책 파일의 각 규칙에 매칭 |
| **6** | 팔로알토 예외처리 | 팔로알토 정책 특성에 따른 예외 조건 반영 |
| **7** | 시큐아이 예외처리 | 시큐아이 정책 특성에 따른 예외 조건 반영 |
| **8** | 중복정책 분류 | 중복 정책을 공지/삭제/유지 대상으로 분류 |
| **9** | 중복 상태 업데이트 | 분류된 작업 구분을 원본 정책 파일에 표시 |
| **10** | 히트카운트 병합 | HA 구성(Pri/Sec) 장비의 미사용 정보를 하나로 통합 |
| **11** | 미사용 정보 업데이트 | 최종 통합된 미사용 정보를 정책 파일에 반영 |
| **12** | 미사용 예외 업데이트 | 중복 정책 등에 의한 미사용 예외 건 처리 |
| **13** | 공지 파일 생성 | 정리 대상별(기간만료, 장기미사용 등) 최종 공지용 엑셀 생성 |
| **14** | 자동연장 점검 | 동일 신청번호의 기간 연장 여부 최종 확인 |

## ⚙️ 설정 가이드 (`fpat.yaml`)

본 모듈은 `exceptions` 섹션을 통해 분석 제외 대상을 관리합니다.

```yaml
exceptions:
  request_ids:
    - id: "PS-2024-0001"
      reason: "특수 업무용"
      until: "2025-12-31"
  policy_rules:
    - pattern: "^BACKUP_.*"
      reason: "백업 예외"
```

---
*참고: 정책 삭제 프로세스는 향후 파이프라인 엔진 도입을 통해 더욱 간소화될 예정입니다.*
