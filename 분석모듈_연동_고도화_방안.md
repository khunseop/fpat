# 방화벽 모듈 연동 및 정책 분석 고도화 방안

본 문서는 `firewall_module`(데이터 추출)과 `firewall_analyzer`(정책 분석) 간의 데이터 규격을 통일하고, 객체 해석(Resolution) 기반의 정밀 분석 체계를 구축하기 위한 분석 결과 및 설계 지침입니다.

---

## 1. 현행 체계 분석 및 한계점

### 1.1 데이터 규격 현황
현재 각 벤더별 콜렉터(PaloAlto, NGF, MF2)는 다음과 같은 통합 규격의 DataFrame을 생성합니다.

- **Security Rules**: `Rule Name`, `Source`, `Destination`, `Service`, `Action`, `Enable` (Source/Dest/Service는 객체 이름 포함)
- **Address Objects**: `Name`, `Type`, `Value` (Type: `ip-netmask`, `ip-range` 등)
- **Service Objects**: `Name`, `Protocol`, `Port`
- **Group Objects**: `Group Name`, `Entry` (멤버 리스트)

### 1.2 주요 한계점
1. **텍스트 기반 비교의 한계**: `RedundancyAnalyzer`가 객체의 이름(Name)을 기준으로 텍스트 매칭을 수행하므로, 이름이 다르지만 실제 IP 대역이 같은 경우 중복 탐지가 불가능합니다.
2. **논리적 포함 관계 미지원**: IP 범위나 서브넷 간의 수학적 포함 관계(Containment)를 계산하지 못하고 단순 일치 여부만 판단합니다.
3. **연동 프로세스 분절**: 추출된 데이터프레임을 다시 분석기에 수동으로 매핑해야 하는 구조적 불편함이 존재합니다.

---

## 2. "완전한 분석"을 위한 아키텍처 설계

### 2.1 데이터 규격 표준화 (Standardization)
추출 모듈(`firewall_module`)의 출력 규격을 분석 모듈의 표준 입력 규격으로 채택합니다. 모든 분석 로직은 위 1.1절에서 정의된 컬럼명을 참조하도록 통일합니다.

### 2.2 고도화된 워크플로우: Extraction -> Resolution -> Analysis
단순 텍스트 치환을 넘어 논리적 비교가 가능한 구조로 개편합니다.

#### [Step 1] Full Resolution (PolicyResolver 고도화)
`PolicyResolver`가 정책 내의 모든 객체 이름을 실제 값으로 해소합니다.
- **Input**: 정책, 주소, 서비스, 그룹 등 전체 데이터프레임.
- **Process**:
    - 그룹 객체의 재귀적 풀기 (Recursive Group Expansion).
    - 객체 이름을 실제 IP/포트 값으로 치환.
    - `Extracted Source`, `Extracted Destination`, `Extracted Service` 컬럼 생성.

#### [Step 2] Logical Redundancy Analysis (정밀 분석)
`RedundancyAnalyzer`가 치환된 값을 바탕으로 수학적 포함 관계를 검사합니다.
- **집합 비교 엔진**: `ipaddress` 라이브러리를 활용하여 서브넷/범위 간의 포함 여부 판단.
- **중복 판단 공식**:
    - $Policy_A.Src \subseteq Policy_B.Src$
    - $Policy_A.Dst \subseteq Policy_B.Dst$
    - $Policy_A.Svc \subseteq Policy_B.Svc$
    - 위 조건을 모두 만족하고 Action이 동일할 경우 $Policy_A$는 $Policy_B$에 의해 중복됨으로 판단.

## 3. 통합 API 설계 (Proposed)

분석 모듈에 콜렉터 인스턴스를 직접 인자로 받아 "추출부터 분석까지" 원스톱으로 처리하는 통합 진입점 도입을 제안합니다.

```python
def run_integrated_analysis(collector: FirewallInterface):
    # 1. 전역 데이터 추출
    rules = collector.export_security_rules()
    objects = {
        "address": collector.export_network_objects(),
        "addr_group": collector.export_network_group_objects(),
        "service": collector.export_service_objects(),
        "svc_group": collector.export_service_group_objects()
    }

    # 2. 객체 해소 (Resolver)
    resolver = PolicyResolver()
    resolved_df = resolver.resolve(rules, **objects)

    # 3. 논리적 정밀 분석 실행
    analyzer = RedundancyAnalyzer()
    return analyzer.analyze_logical(resolved_df)
```

## 4. 향후 실행 과제
1. **`PolicyResolver.resolve()` 수정**: 모든 추출 데이터프레임을 인자로 받아 일괄 처리하도록 인터페이스 개선.
2. **수학적 비교 로직 추가**: `PolicyFilter` 내의 IP 중첩 검사 로직을 `RedundancyAnalyzer`에 통합.
3. **분석기 인터페이스 통일**: 벤더별 컬럼 매핑 로직을 설정 파일(`fpat.yaml`) 또는 콜렉터 속성으로부터 가져오도록 개선.
